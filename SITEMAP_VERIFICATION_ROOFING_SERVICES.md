## Issue Resolution
You were correct — the initial sitemap generation script was NOT including `priority` and `changefreq` tags. I've fixed this and rebuilt.

## What Was Fixed

### Problem
The sitemap generation script (`scripts/generate-sitemap.mjs`) was only outputting `<loc>` tags without including the `priority` and `changefreq` metadata that was defined in `sheetSitemap.ts`.

### Solution
Updated the XML generation code (lines 337-349) to include all metadata:

```javascript
const urlEntries = dedupedEntries.map(entry => {
  const url = `${CANONICAL_DOMAIN}${entry.path}`;
  let urlEntry = `  <url>\n`;
  urlEntry += `    <loc>${url}</loc>\n`;
  if (entry.priority !== undefined) {
    urlEntry += `    <priority>${entry.priority}</priority>\n`;
  }
  if (entry.changefreq) {
    urlEntry += `    <changefreq>${entry.changefreq}</changefreq>\n`;
  }
  urlEntry += `  </url>`;
  return urlEntry;
}).join('\n');
```
